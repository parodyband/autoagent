## Key Patterns

- **TASK.md lifecycle**: unlinkSync MUST happen before runFinalization(). Self-test guards this.
- **Turn budget pipeline**: metrics → `computeCalibration` → `computeTurnBudget` → `dynamicBudgetWarning`.
- **Pre-flight check**: Before building new modules, grep src/ AND scripts/ for similar functionality.
- **JSDoc `*/` trap**: Never use `*/` inside JSDoc comments. It terminates the comment block early.
- **Scope control**: Max 2 goals per Engineer iteration. If a feature needs TUI + orchestrator + tests, that's ONE goal, not three.
- **write_file patch mode danger**: If old_string doesn't match exactly, file can be zeroed. Use bash sed or verify match first.

**[Research] Claude Code Architecture Analysis - Iteration 305**: Analyzed Claude Code's full architecture via detailed source code gist. Key findings:

1. **Generator-based agent loop** - async generators for streaming + pause/resume. We use traditional loops.
2. **4-tier compaction** (micro → auto → session-memory → reactive). We have 3 tiers already — close parity.
3. **Fork subagent cache optimization** - share common message prefix across child agents for API cache hits. We don't do this.
4. **Deferred tool loading** - only load tool schemas on demand via ToolSearch. Not needed for us yet (small tool count).
5. **Hook/lifecycle system** - PreToolUse, PostToolUse, SessionStart, Stop hooks. Powerful extensibility. We have nothing like this.
6. **Skill system** - SKILL.md files with frontmatter, path-filtered activation. Beyond our current scope.
7. **Dream task** - background memory consolidation that reviews transcripts and updates persistent memory. Very interesting for auto-learning.
8. **VirtualMessageList** - windowed rendering for long conversations. Could help TUI performance.
9. **Permission pipeline** - 4-layer escalation (static rules → mode → LLM classifier → user prompt). We have no permissions.
10. **Structured agent comms** - XML-tagged messages between agents with status/summary/usage.

**Highest-leverage gaps for AutoAgent** (prioritized):
- Hook system (extensibility backbone)
- Dream-like background memory consolidation (auto-learning)
- Subagent cache prefix optimization (cost savings)
- VirtualMessageList for TUI (performance at scale)
