**[Research] Claude Code Architecture Deep Dive — Key Patterns for AutoAgent**: From analysis of Claude Code source (gist/yanchuk, March 2026):

**High-priority patterns we should adopt:**
1. **Prompt caching breakpoints** — Strategic `cache_control: { type: "ephemeral" }` placement on system prompt and message boundaries. We're likely paying full price for repeated context on every API call. HUGE cost savings potential.
2. **Fork subagent for cache sharing** — When spawning subagents, construct message history with shared prefix so API cache hits. All children share ~100K cached (cheap) instead of ~100K uncached each.
3. **Dream Task (background memory consolidation)** — During idle time, agent reviews session transcripts, identifies patterns/decisions, updates memory files. No user disruption.
4. **Hook system (PreToolUse/PostToolUse)** — Lifecycle hooks users can extend via shell commands, enabling custom linting before writes, auto-formatting after edits, audit logging.
5. **Deferred tool loading** — Only load tool names in system prompt, fetch full schemas on demand via ToolSearch. Saves context when tool count grows.
6. **Virtual message list** — Windowed rendering for TUI performance in long conversations.

**Patterns we already have (good):**
- Streaming agent loop (generator-like)
- Tiered compaction (micro/T1/T2) — Claude Code has 4 tiers, we have 3
- Parallel tool execution
- Sub-agent delegation
- File watcher integration
