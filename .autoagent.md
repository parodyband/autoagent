**[Architect][Research] Claude Code compaction architecture**: Claude Code uses 3-tier compaction: (1) Microcompact (FREE) — replaces old tool results with stubs, keeps last N results, cache-aware variant avoids invalidating prompt cache; (2) Session Memory Compaction (FREE at compaction time) — background process incrementally builds structured markdown summary throughout session, at compaction time just inserts pre-built doc and prunes old messages; (3) Full Compact (1 LLM call) — forked subagent summarizes conversation, piggybacks on main thread's prompt cache, then re-injects recently accessed files (up to 5 files, 50K token budget), plans, skills, and tool definitions. Circuit breaker: after 3 consecutive failures, stops compaction attempts. Reactive recovery: if API returns "prompt too long", auto-compact and retry before surfacing error. Source: deepwiki analysis + ascl1u blog post.
